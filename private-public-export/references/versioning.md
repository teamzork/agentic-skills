# Release Versioning (SemVer)

Use semantic versioning: MAJOR.MINOR.PATCH.

## When to bump

- PATCH: bug fixes, dependency bumps, docs, or small internal changes.
- MINOR: new features or new capabilities that remain backward-compatible.
- MAJOR: breaking changes in CLI, data formats, config, or behavior.

## Examples

- v0.1.0 -> v0.1.1: dependency update or small fix.
- v0.1.1 -> v0.2.0: new parser, new CLI flag, or extra export format.
- v0.9.4 -> v1.0.0: stable, publicly supported interface.
