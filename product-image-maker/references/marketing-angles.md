# Marketing Angles Catalog

A concept direction is the *idea* a marketing image communicates — not its color. The five options you generate should span different families here so the human is choosing between genuinely different stories, not five tints of one layout.

Pick by fit: process products suit Flow/Loop; transformation products suit Before/After; feature-rich products suit Bento; bold positioning suits Big Claim. When unsure, the safe spread is **Product Hero + Flow + Before/After + Bento + Big Claim** — one from each family.

Every angle below renders in the **app's own design system** (fonts, palette, shapes, signature components). The sketches show layout, not style.

---

## 1. Product Hero  *(family: show-the-product)*
Headline + one supporting line on one side, a faithful faux app card showing the product mid-use on the other. The most reliable, most common hero. Reads as "here's the thing, and here's it working."

```
┌─────────────────────────────────────────────┐
│ ● Brandmark                                  │
│                          ┌────────────────┐  │
│  Big bold headline       │  TODAY'S PROMPT │  │
│  with one teal accent.   │  Faux app UI…   │  │
│                          │   ( ● record )  │  │
│  ✓ benefit one           │  ▸ result chip  │  │
│  ✓ benefit two           └────────────────┘  │
│  ✓ benefit three                             │
│  …small reassuring footer line               │
└─────────────────────────────────────────────┘
```
Copy: one headline (≤6 words), 2–3 checkmark benefits, optional quiet footer. Best when the UI itself is attractive and self-explanatory.

## 2. How-It-Works Flow  *(family: process)*
Three or four numbered steps with arrows. Great for products whose value is a clear pipeline. Teaches the mechanism at a glance.

```
┌─────────────────────────────────────────────┐
│  How it works                                │
│                                              │
│  ①  Speak    →  ②  We analyze  →  ③ You learn│
│  [icon card]    [icon card]      [icon card] │
│                                              │
│  one-line tagline under the rail             │
└─────────────────────────────────────────────┘
```
Copy: 3–4 short verb-led steps. Keep each step to a noun phrase + one clause.

## 3. Before / After  *(family: transformation)*
Split canvas: the "before" pain state vs the "after" the product delivers. Sells outcomes, not features.

```
┌──────────────────────┬──────────────────────┐
│  BEFORE              │  AFTER                │
│  messy / stuck       │  clear / fluent       │
│  [dim, muted]        │  [bright, on-brand]   │
│  "what it felt like" │  "what it feels like" │
└──────────────────────┴──────────────────────┘
```
Copy: parallel phrasing across the divide. Desaturate the before; let the after carry the brand accent.

## 4. The Loop  *(family: process)*
A cyclical diagram for habit-forming or iterative products (practice loops, feedback cycles, review cadences). Communicates "this compounds over time."

```
┌─────────────────────────────────────────────┐
│              ┌───────────────┐               │
│        ┌────▶│   step A      │────┐          │
│        │     └───────────────┘    ▼          │
│   ┌─────────┐               ┌──────────┐     │
│   │ step D  │◀──────────────│  step B  │     │
│   └─────────┘               └──────────┘     │
│        ▲      ┌───────────┐      │           │
│        └──────│  step C   │◀─────┘           │
│               └───────────┘                  │
└─────────────────────────────────────────────┘
```
Copy: 3–5 nodes, each a short phrase. Headline names the loop ("The practice loop").

## 5. Bento Grid  *(family: feature-showcase)*
Tiles of different sizes, each a feature or proof point. Modern, dense, premium. Good when the product has several distinct strengths.

```
┌───────────────┬───────────┬───────────┐
│  hero tile    │  feature  │  metric   │
│  (2×)         ├───────────┼───────────┤
│               │  feature  │  feature  │
└───────────────┴───────────┴───────────┘
```
Copy: one anchor tile with the headline; supporting tiles each carry a single feature label + tiny detail. Vary tile sizes for rhythm.

## 6. Big Claim / Editorial  *(family: positioning)*
One bold typographic statement, minimal everything else. Magazine-cover energy. Best for confident positioning where the words do the work.

```
┌─────────────────────────────────────────────┐
│                                              │
│     Your spontaneous speech,                 │
│     gently taught.                           │
│                                              │
│              ● small brandmark               │
└─────────────────────────────────────────────┘
```
Copy: one sentence, set large in the display face, one accent color on the key phrase. Heavy negative space.

## 7. Social Proof / Testimonial  *(family: trust)*
A quote with attribution and a result metric. Best once there are real users and real numbers — never fabricate quotes or stats.

```
┌─────────────────────────────────────────────┐
│  "short punchy quote about the outcome."     │
│                                              │
│   ◐ Name, role            ↑ 3× metric        │
└─────────────────────────────────────────────┘
```
Copy: real quote, real attribution, real number. If you don't have them, pick a different angle.

## 8. Annotated UI / Feature Spotlight  *(family: show-the-product)*
A single product screen with callout lines pointing to 2–3 key parts. Best when one screen captures the whole value and the details reward a closer look.

```
┌─────────────────────────────────────────────┐
│   ╭── callout: "live feedback"               │
│   │   ┌─────────────────────────┐            │
│   │   │   real app screen        │── callout │
│   ▼   │                          │   "one tap"│
│       └─────────────────────────┘            │
└─────────────────────────────────────────────┘
```
Copy: 2–3 short callouts, each pointing at a real UI element. Don't over-annotate.

---

## Choosing the five
- Span **at least 3 families** (show-the-product, process, transformation, feature-showcase, positioning, trust).
- Include at least one Product Hero or Annotated UI — buyers want to see the actual thing.
- Only use Testimonial if you have real proof.
- If the product is pre-launch with one strong screen: Hero + Annotated UI + Flow + Before/After + Big Claim.
